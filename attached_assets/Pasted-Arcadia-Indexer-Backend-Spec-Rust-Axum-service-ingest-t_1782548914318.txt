1.Overview & Responsibilities
Arcadia lets traders build a verified, permanent, risk-adjusted onchain reputation from real trades, and lets investors fund proven traders and share in their profits. Three technical surfaces are built by three engineers in eight weeks: the Solana program (the vault & settlement logic), this indexer/backend, and the frontend.

This document specs the indexer / backend. It is a single Rust binary (workspace of crates) running several tokio tasks behind one Axum HTTP server. It consumes the program's events over Yellowstone gRPC, persists them to Postgres/TimescaleDB, computes the Arcadia Score and Capacity, serves the /v1 API the frontend reads, and acts as the program's oracle_authority - pulling real market prices to co-sign simulated trades and pushing the computed capacity ceiling onchain.

MVP scope (locked): MVP plus basic investor deposits. Score uses absolute bootstrap curves (no percentile cohort yet). Capacity ceiling is computed and pushed onchain. Trades are simulated on devnet - the backend is the oracle_authority that pulls real prices and co-signs record_trade. Identity is SIWS only.
Where integer-only applies, and where it does not. This indexer is offchain. The "integers only" rule belongs to the onchain program, not here. The scoring engine is free to use rust_decimal (preferred for money/PnL/NAV) or f64 for the score math itself (Sortino, Calmar, Ulcer, logistic confidence, exponential capacity, interpolation) - none of that math has to be integer. The one hard constraint: the single capacity value the indexer pushes onchain via set_capacity must be a plain u64 in USDC minor units (6 decimals), because the program stores capacity_cap_usd: u64. Convert/round to u64 minor units only at that onchain boundary. House preference for money columns and accruals (deposits, shares, PnL, fees, NAV) remains rust_decimal to avoid float drift in dollar figures shown to users.

2.Architecture
One process, one Tokio runtime. Five logical components run as supervised tasks; the Axum API and the workers share a PgPool, a Redis connection, and a read-only config. Each worker is restarted with backoff if it panics.

// main.rs - supervised task fan-out
#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let cfg  = Config::from_env()?;
    let pg   = PgPoolOptions::new().max_connections(20).connect(&cfg.database_url).await?;
    let rds  = redis::Client::open(cfg.redis_url.clone())?;
    sqlx::migrate!().run(&pg).await?;

    let ctx = AppCtx { pg, rds, cfg };

    // each worker = its own restart-with-backoff supervisor
    let ingest   = supervise("ingest",   ingest::run(ctx.clone()));
    let score    = supervise("score",    score::run(ctx.clone()));    // hourly
    let capacity = supervise("capacity", capacity::run(ctx.clone()));
    let oracle   = supervise("oracle",   oracle::run(ctx.clone()));
    let api      = api::serve(ctx.clone());                          // Axum

    tokio::try_join!(ingest, score, capacity, oracle, api)?;
    Ok(())
}

4.Database Schema (Postgres + TimescaleDB)
All USD / share / PnL columns are NUMERIC and mapped to rust_decimal::Decimal. The program emits integers in USDC minor units (USDC has 6 decimals, so 1 USDC = 1,000,000 minor units) and NAV scaled by 1e6 (amount_usd:u64 in USDC minor units, nav_per_share:u64 scaled 1e6); we record the raw scaled value and the decoded decimal. trade and equity_point are TimescaleDB hypertables (time partitioned on closed_at / day). The onchain TraderProfile account is the vault (PDA, the program's signing authority); we mirror it in trader_profile. Shares are data, not tokens: an investor funding multiple traders holds one investor_position row per trader profile, all hanging off a single investor_account. The mirrored capacity_cap_usd matches the program's capacity_cap_usd: u64 (USDC minor units); we keep it as NUMERIC here but it is always a whole-number minor-unit value.

5.Event Decoding
The program is an Anchor program. Each emitted event is logged as a base64 Program data: line whose bytes begin with an 8-byte discriminator = first 8 bytes of sha256("event:<EventName>"), followed by the borsh-serialized struct. The ingest worker subscribes via Yellowstone gRPC, matches the discriminator, then borsh-decodes the remainder.

// decode/discriminators.rs - precomputed at build time
use sha2::{Digest, Sha256};

fn disc(name: &str) -> [u8; 8] {
    let h = Sha256::digest(format!("event:{name}").as_bytes());
    h[..8].try_into().unwrap()
}

// borsh-mirror structs (must match the program's field order EXACTLY)
#[derive(BorshDeserialize)]
struct TradeClosed {
    profile: Pubkey, trader: Pubkey, market: String, direction: u8,
    size_usd: u64, leverage_x100: u16, entry_px: u64, exit_px: u64,
    realized_pnl: i64, fees_usd: u64, was_liquidated: bool,
    opened_at: i64, closed_at: i64,
}
// ...ProfileInitialized, InvestorInitialized, Deposited,
//    WithdrawRequested, Withdrawn, Settled, ProfitWithdrawn likewise

enum ArcadiaEvent {
    ProfileInitialized(..), InvestorInitialized(..), Deposited(..),
    WithdrawRequested(..), Withdrawn(..), TradeClosed(TradeClosed),
    Settled(..), ProfitWithdrawn(..),
}

fn decode_event(bytes: &[u8]) -> Option<ArcadiaEvent> {
    let (d, mut payload) = bytes.split_at(8);
    match d {
        x if x == DISC_PROFILE_INIT  => Some(ArcadiaEvent::ProfileInitialized(
            ProfileInitialized::deserialize(&mut payload).ok()?)),
        x if x == DISC_INVESTOR_INIT => Some(ArcadiaEvent::InvestorInitialized(
            InvestorInitialized::deserialize(&mut payload).ok()?)),
        x if x == DISC_DEPOSITED     => Some(ArcadiaEvent::Deposited(
            Deposited::deserialize(&mut payload).ok()?)),
        x if x == DISC_WITHDRAW_REQ  => Some(ArcadiaEvent::WithdrawRequested(
            WithdrawRequested::deserialize(&mut payload).ok()?)),
        x if x == DISC_WITHDRAWN     => Some(ArcadiaEvent::Withdrawn(
            Withdrawn::deserialize(&mut payload).ok()?)),
        x if x == DISC_TRADE         => Some(ArcadiaEvent::TradeClosed(
            TradeClosed::deserialize(&mut payload).ok()?)),
        x if x == DISC_SETTLED       => Some(ArcadiaEvent::Settled(
            Settled::deserialize(&mut payload).ok()?)),
        x if x == DISC_PROFIT_WD     => Some(ArcadiaEvent::ProfitWithdrawn(
            ProfitWithdrawn::deserialize(&mut payload).ok()?)),
        _ => None,
    }
}

Idempotency & resume
Every write is INSERT ... ON CONFLICT (signature, event_index) DO NOTHING - replays from Yellowstone never double-count.
A cursor row stores the last fully-processed slot; on reconnect the subscription resumes from last_slot and re-decodes (harmless thanks to the conflict clause).
Minor-unit decoding is centralized: fn usdc_minor(v:u64)->Decimal { Decimal::from(v) / dec!(1_000_000) } (USDC has 6 decimals, so divide by 1,000,000), price scale likewise - one place, unit-tested.
// workers/ingest.rs - the subscribe loop (sketch)
let mut stream = client.subscribe(program_filter(PROGRAM_ID)).await?;
while let Some(update) = stream.next().await {
    for (sig, slot, idx, raw) in extract_program_data(&update) {
        if let Some(ev) = decode_event(&raw) {
            persist(&ctx.pg, sig, slot, idx, ev).await?;  // ON CONFLICT DO NOTHING
        }
    }
    save_cursor(&ctx.pg, slot).await?;
}
6.Reconstruction, TWR & Metrics
Reconstruction is trivial here. The program already emits clean, closed TradeClosed events - there is no open/close pairing, fill stitching, or position bookkeeping to do. We persist trades verbatim. The only non-trivial reconstruction is the equity curve.
Time-weighted return (TWR) equity curve
Critical: you MUST strip deposits and withdrawals out of returns. If you compute equity as raw AUM, a deposit looks like a profit and a withdrawal looks like a loss - corrupting every downstream metric and the score. Use TWR: chain-link daily sub-period returns where flows happen at period boundaries.
For each vault, walk days from inception. Within a day, sort flows and trades by time. The day's return strips external cash flows:

// scoring/twr.rs - build daily TWR curve from flows + trades
fn build_equity_curve(flows: &[Flow], trades: &[Trade]) -> Vec<EquityPoint> {
    let mut aum   = Decimal::ZERO;      // dollar AUM, includes flows
    let mut index = dec!(1.0);          // TWR index, base 1.0
    let mut out   = Vec::new();

    for day in inception..=today {
        let start_aum   = aum;
        let net_flow    = sum_flows_on(day, flows);   // +deposits −withdrawals
        let pnl         = sum_pnl_on(day, trades);    // realized PnL − fees

        // time-weighted daily return: PnL over (capital actually at work),
        // excluding the flow that arrived this period (Modified Dietz, flow at mid)
        let denom = start_aum + net_flow / dec!(2);
        let daily_ret = if denom.is_zero() { Decimal::ZERO } else { pnl / denom };

        index *= (dec!(1) + daily_ret);
        aum    = start_aum + net_flow + pnl;          // roll forward dollar AUM

        out.push(EquityPoint { day, nav: index, aum_usd: aum, daily_ret });
    }
    out
}
The nav column of equity_point is the TWR index. return_30d / 90d / 1y are index[t]/index[t-N] − 1 for display only. The score uses the full-history daily-return series, never a window.

Metrics (all rust_decimal)
Metric	Definition
Sortino	mean(daily_ret) / downside_deviation(daily_ret), annualized (×√365). Downside dev uses only negative returns vs 0 target.
Calmar	annualized return / |max drawdown|.
MaxDrawdown	max peak-to-trough decline of the TWR index, as a positive fraction.
Ulcer index	√(mean(drawdown%²)) over the curve - penalizes deep & long drawdowns.
LiqRate	count(trades where was_liquidated) / total trades.
%Profitable	fraction of trades with realized_pnl > 0 (and %profitable months from monthly buckets, for consistency).
AvgLeverage / LevVol	mean and stdev of leverage_x across trades.
ExcessReturn	vault annualized return − SOL buy-&-hold return over the same span.
ReturnCoeffVar	stdev(monthly returns) / |mean(monthly returns)|.
SurvivalDays / DistinctMarkets / RegimesSurvived	days_active; count(distinct market); count of market regimes (bull/bear/chop buckets) the vault traded through.
rust_decimal sqrt: use a fixed-iteration Newton's method on Decimal (the crate has no native sqrt) for downside deviation, Ulcer, and annualization √365. Keep it in scoring/mathx.rs, unit-tested against known values.
7.Scoring & Capacity Engine
Implemented exactly as specified. All inputs are full-history metrics (never a rolling window). Pure functions in crates/scoring, unit-tested. This engine is offchain, so the math here uses rust_decimal (or f64 where convenient); it does not need to be integer. Only the capacity number eventually pushed via set_capacity is converted to a u64 in USDC minor units. Every constant used below is also collected in the Numbers & Formulas reference (§8).

Prior: the Bayesian shrinkage prior is mu = 400 (the value the blended score is pulled toward when confidence is low). The confidence interval half-width is CI = ±125 * (1 - C): at C = 0 the band is ±125; at C = 1 it collapses to ±0.
Top-level score
// SCORE = G * [ C*Q + (1-C)*400 ]   in 0..1000     prior mu = 400     CI = +/- 125*(1-C)
fn score(m: &Metrics, ctx: &Context) -> ScoreResult {
    let q = quality(m);                     // 0..1000
    let c = confidence(ctx);                 // 0..1
    let g = integrity(m, ctx);               // 0..1 (penalty product)
    let raw = c*q + (dec!(1)-c)*dec!(400);
    let score = g * raw;                     // 0..1000
    let ci = dec!(125) * (dec!(1)-c);
    ScoreResult { score, confidence: c, q, g,
                  ci_lo: (score-ci).max(dec!(0)),
                  ci_hi: (score+ci).min(dec!(1000)),
                  tier: tier_of(score) }
}
Quality Q (weighted sub-pillars)
Pillar weights (must sum to 1.0): Q = 0.30*P_perf + 0.28*P_risk + 0.20*P_consistency + 0.12*P_execution + 0.10*P_track. Sub-weights:

P_perf = 0.45 Sortino + 0.30 Calmar + 0.25 ExcessReturn
P_risk = 0.40 (-MaxDD) + 0.25 (-Ulcer) + 0.20 (-LiqRate) + 0.15 (-LevVol)
P_consistency = 0.50 (%ProfitableMonths) + 0.30 (-ReturnCoeffVar) + 0.20 (RegimesSurvived)
P_execution = 0.40 (-Slippage) + 0.30 (TP-SL adherence) + 0.30 (CapitalEfficiency)
P_track = 0.50 (SurvivalDays) + 0.50 (DistinctMarkets)
fn quality(m: &Metrics) -> Decimal {
    let p_perf =
        dec!(0.45)*map_sortino(m.sortino)
      + dec!(0.30)*map_calmar(m.calmar)
      + dec!(0.25)*map_excess(m.excess_return);          // vs SOL-hold

    let p_risk =
        dec!(0.40)*map_maxdd(m.max_dd)
      + dec!(0.25)*map_ulcer(m.ulcer)
      + dec!(0.20)*map_liqrate(m.liq_rate)
      + dec!(0.15)*map_levvol(m.lev_vol);

    let p_cons =
        dec!(0.50)*map_pct_months(m.pct_profitable_months)
      + dec!(0.30)*map_inv(m.return_coeff_var)
      + dec!(0.20)*map_regimes(m.regimes_survived);

    // execution: sim default-neutral (500) where data is absent
    let p_exec =
        dec!(0.40)*m.slippage_score.unwrap_or(dec!(500))
      + dec!(0.30)*m.tpsl_adherence.unwrap_or(dec!(500))
      + dec!(0.30)*m.capital_efficiency.unwrap_or(dec!(500));

    let p_track =
        dec!(0.50)*map_survival(m.survival_days)
      + dec!(0.50)*map_markets(m.distinct_markets);

    dec!(0.30)*p_perf + dec!(0.28)*p_risk + dec!(0.20)*p_cons
  + dec!(0.12)*p_exec + dec!(0.10)*p_track
}
Bootstrap absolute curves m()
Piecewise-linear interpolation between the anchors below. Implement one generic interp(x, &[(knot, out)]) helper.

Metric	Anchors (input → 0–1000)
m(Sortino)	≤0→0, 1→400, 2→650, 3→820, ≥4→950
m(Calmar)	≤0→0, 1→450, 2→700, ≥3→900
m(−MaxDD)	≥40%→0, 30%→350, 20%→600, 10%→850, <5%→1000
m(−LiqRate)	≥30%→0, 15%→400, 5%→700, 0%→1000
m(avgLeverage)	≥10x→150, 8x→400, 5x→700, ≤3x→1000
m(%profitable)	linear 0%→0 … 100%→1000
Ulcer, LevVol, ExcessReturn, ReturnCoeffVar, RegimesSurvived, SurvivalDays, DistinctMarkets use analogous monotone curves with the same interp() helper (sensible MVP anchors; tune later). map_inv = curve on the negated/inverted metric.

fn interp(x: Decimal, knots: &[(Decimal, Decimal)]) -> Decimal {
    if x <= knots[0].0 { return knots[0].1; }
    for w in knots.windows(2) {
        let ((x0,y0),(x1,y1)) = (w[0], w[1]);
        if x <= x1 {
            let t = (x - x0) / (x1 - x0);
            return y0 + t*(y1 - y0);
        }
    }
    knots.last().unwrap().1
}
fn map_sortino(s: Decimal) -> Decimal {
    interp(s, &[(dec!(0),dec!(0)),(dec!(1),dec!(400)),(dec!(2),dec!(650)),
                (dec!(3),dec!(820)),(dec!(4),dec!(950))])
}
Confidence C (logistic)
// C = 1 / (1 + e^(-z))
// z = 0.55*ln(1+trades) + 0.65*ln(1+days_active) + 0.35*ln(1+capital_at_risk_usd)
//   + 0.60*regimes_survived + 0.30*ln(1+distinct_markets) - 7.0
fn confidence(c: &Context) -> Decimal {
    let z = dec!(0.55)*ln1p(c.trades)
          + dec!(0.65)*ln1p(c.days_active)
          + dec!(0.35)*ln1p(c.capital_at_risk_usd)
          + dec!(0.60)*Decimal::from(c.regimes_survived)
          + dec!(0.30)*ln1p(c.distinct_markets)
          - dec!(7.0);
    sigmoid(z)   // 0..1
}
Integrity G (penalty product)
// G = Π (1 - penalty_i)   - MVP detectors only
fn integrity(m: &Metrics, c: &Context) -> Decimal {
    let mut g = dec!(1.0);
    // small-sample gate: too few trades → cap trust
    if c.trades < 10 { g *= dec!(0.70); }
    if c.trades < 3  { g *= dec!(0.50); }
    // obvious self-trade: investor wallet == trader wallet on flows
    if m.self_trade_detected { g *= dec!(0.60); }
    g
}
Capacity engine
Explicit formulas: ceiling_usd = 2500 * e^((score * C) / 140); deployable_usd = min(ceiling_usd, earned_size_usd) * liquidity_factor; liquidity_factor = majors 1.0 / mid 0.6 / low 0.3. These run offchain in Decimal/f64. The chosen deployable_usd is then rounded down and pushed onchain as a u64 in USDC minor units via set_capacity.

// ceiling_usd    = 2500 * e^((score * C) / 140)
// deployable_usd = min(ceiling_usd, earned_size_usd) * liquidity_factor
// liquidity_factor: majors 1.0, mid 0.6, low 0.3
enum Liquidity { Majors, Mid, Low }
fn liq_factor(l: Liquidity) -> Decimal {
    match l { Liquidity::Majors=>dec!(1.0), Liquidity::Mid=>dec!(0.6), Liquidity::Low=>dec!(0.3) }
}
fn capacity(score: Decimal, c: Decimal, earned: Decimal, liq: Liquidity) -> Capacity {
    let ceiling   = dec!(2500) * exp((score * c) / dec!(140));
    let deployable = ceiling.min(earned) * liq_factor(liq);
    Capacity { ceiling, deployable }
}

// at the onchain boundary only: Decimal USD -> u64 USDC minor units (6 decimals)
fn to_minor_units_u64(usd: Decimal) -> u64 {
    (usd * dec!(1_000_000)).floor().to_u64().unwrap_or(0)   // program: capacity_cap_usd: u64
}

9.Oracle-Authority Service
The backend holds the program's oracle_authority keypair. It does two onchain jobs.

D1 - Co-sign simulated trades with real prices
A sim-trade scheduler (or trader-submitted intent queue) produces trade requests. For each, the signer pulls the real entry/exit price from the price feed, then builds and co-signs the record_trade(...) instruction handler. The program records the trade; the indexer later sees the resulting TradeClosed event - the loop closes through the chain, not in memory.

// workers/oracle.rs
async fn handle_sim_trade(ctx: &AppCtx, req: SimTradeReq) -> Result<Signature> {
    let px = prices::current(&ctx.cfg, &req.market).await?;     // real Pyth/REST price
    let entry = px.scaled();                                       // u64, fixed scale
    let exit  = prices::at(&ctx.cfg, &req.market, req.close_ts).await?.scaled();

    let ix = chain::record_trade_ix(
        &ctx.cfg.program_id, &req.profile, &ctx.oracle.pubkey(),
        &req.market, req.direction, req.size_usd, req.leverage_x100,
        entry, exit, /* realized_pnl computed onchain */);

    let tx = build_signed(ix, &[&ctx.oracle], recent_blockhash).await?;
    let sig = ctx.rpc.send_and_confirm_transaction(&tx).await?;
    Ok(sig)
}
D2 - Push the dynamic capacity onchain
The indexer pushes the dynamic capacity each scoring run. The capacity worker watches new score_snapshot rows; when capacity or tier moves past a threshold (e.g. ±10% or a tier change), it sends the set_capacity(cap_usd, score_tier) instruction handler signed by the oracle authority, then records the resulting tx signature on the trader_profile row. cap_usd is encoded as a u64 in USDC minor units (the offchain decimal ceiling, rounded down to whole minor units at this boundary); score_tier is a u8 (0=Verified, 1=Established, 2=Advanced, 3=Elite).

async fn push_capacity(ctx:&AppCtx, profile:&str, cap_usd:u64, tier:Tier) -> Result<Signature> {
    let ix  = chain::set_capacity_ix(&ctx.cfg.program_id, profile,
                                    &ctx.oracle.pubkey(), cap_usd, tier as u8);
    let tx  = build_signed(ix, &[&ctx.oracle], recent_blockhash).await?;
    ctx.rpc.send_and_confirm_transaction(&tx).await.map_err(Into::into)
}
Price source
Primary: Pyth Hermes price feed via reqwest (SOL/USD and any traded market); fall back to a REST price API.
Prices cached in Redis with a short TTL; GET /v1/prices proxies the same cache.
Convert to the program's fixed price scale in one place; assert non-stale (publish_time within N seconds) before signing.
Signer-key security
The oracle keypair is loaded from an env-injected secret / KMS file path - never committed, never logged. Keypair stays in memory only.
Only the oracle worker can access it (held in its task; not in AppCtx shared state for the API).
Devnet-only key with its own SOL; rotate by updating the program's oracle_authority. Distinct keypairs per environment.
Stale-price and size-bound guards before every signature; refuse to sign on stale or out-of-range inputs.
All signing actions are tracing-logged with tx signature (never the secret) for audit.
10.SIWS Auth & API Keys
Sign-In With Solana. Two steps: client requests a nonce/challenge; client signs it with the wallet; server verifies the ed25519 signature against the pubkey and issues a bearer JWT.

// POST /v1/auth/challenge → { nonce }
async fn challenge(State(ctx): State<AppCtx>) -> Json<ChallengeRes> {
    let nonce = random_b58(24);
    redis_set_ex(&ctx.rds, &format!("siws:{nonce}"), "1", 300).await; // 5-min TTL
    Json(ChallengeRes { nonce })
}

// POST /v1/auth/verify { pubkey, signature, nonce } → { token }
async fn verify(State(ctx): State<AppCtx>, Json(b): Json<VerifyReq>)
        -> Result<Json<TokenRes>, ApiError> {
    if !redis_take(&ctx.rds, &format!("siws:{}", b.nonce)).await {
        return Err(ApiError::Unauthorized);          // nonce unknown/expired/replayed
    }
    let pk  = VerifyingKey::from_bytes(&bs58_decode32(&b.pubkey)?)?;
    let sig = Signature::from_bytes(&bs58_decode64(&b.signature)?);
    let msg = siws_message(&b.pubkey, &b.nonce);    // canonical SIWS statement
    pk.verify_strict(msg.as_bytes(), &sig)
      .map_err(|_| ApiError::Unauthorized)?;        // ed25519-dalek verify

    let token = issue_jwt(&ctx.cfg.jwt_secret, &b.pubkey, 3600*24);
    Ok(Json(TokenRes { token }))
}
Bearer middleware & B2B API key
Bearer JWT: protected routes (e.g. investor account & portfolio, trader-only actions) require Authorization: Bearer <jwt>; an Axum FromRequestParts extractor verifies it and injects the wallet.
